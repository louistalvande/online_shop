== Contexte

Ce document liste l'ensemble des user stories dérivées des fonctions système (FS) définies en SA et allouées en LA/PA. Chaque user story est tracée vers les éléments ARCADIA dont elle est issue.

Les user stories sont regroupées par domaine fonctionnel et ordonnées selon les dépendances d'implémentation (les stories d'infrastructure avant celles de fonctionnalité).

NOTE: Le statut de chaque story est à renseigner dans la colonne « Statut » au fil du développement. Valeurs suggérées : `À faire` / `En cours` / `Terminée`.

== Annulation de commande

_Fonctions ARCADIA : FS-B09, FS-V08, FS-P03 — UCSA-06 — LA : Service Commande / Service Paiement_

[cols="1,3,4,1", options="header"]
|===
| ID | User story | Critères d'acceptation | Statut

| *US-CAN-01*
| En tant qu'acheteur, je peux annuler une commande en attente ou en préparation depuis mon espace commandes sans que le vendeur puisse s'y opposer.
a|
- [ ] Bouton d'annulation disponible sur les commandes aux statuts `en_attente` et `en_préparation`
- [ ] Confirmation obligatoire avant annulation
- [ ] Si paiement par virement : champ « IBAN de remboursement » obligatoire dans le formulaire d'annulation, avec validation du format IBAN
- [ ] Si paiement par carte : aucun champ bancaire requis
- [ ] Annulation effective immédiatement ; stock libéré ; vendeur notifié
| À faire

| *US-CAN-02*
| En tant que vendeur, je suis notifié d'une annulation avant expédition et dois procéder au remboursement si le paiement était par virement.
a|
- [ ] Notification de l'annulation dans le back-office (lecture seule — aucun refus possible)
- [ ] Paiement carte → remboursement déclenché automatiquement par le système via Stripe (FS-P03) ; aucune action requise du vendeur
- [ ] Paiement virement → IBAN de l'acheteur et montant affichés au vendeur pour virement inverse ; commande en `remboursement_virement_en_cours`
| À faire

| *US-CAN-03*
| En tant que vendeur, je peux accepter une annulation après expédition en exigeant le renvoi préalable du colis.
a|
- [ ] Commande passe en statut `en_attente_retour` ; acheteur notifié du renvoi requis
- [ ] Vendeur confirme la réception du colis retourné dans le back-office
- [ ] Remboursement carte → déclenché via Stripe (FS-P03) après confirmation de réception
- [ ] Remboursement virement → commande passe en `remboursement_virement_en_cours` ; IBAN + montant fournis au vendeur pour virement inverse
| À faire

| *US-CAN-04*
| En tant que vendeur, je peux accepter une annulation après expédition sans exiger le retour du colis.
a|
- [ ] Paiement carte → remboursement déclenché immédiatement via Stripe (FS-P03) ; commande passe en `annulée` ; acheteur notifié
- [ ] Paiement virement → commande passe en `remboursement_virement_en_cours` ; IBAN + montant fournis au vendeur pour virement inverse
| À faire

| *US-CAN-05*
| En tant que vendeur, je peux confirmer l'émission d'un virement inverse de remboursement vers l'acheteur.
a|
- [ ] Bouton « Confirmer le virement de remboursement » accessible depuis la commande en statut `remboursement_virement_en_cours`
- [ ] Confirmation → commande passe en `annulée` (FS-V10) ; acheteur notifié que le virement de remboursement a été émis
- [ ] IBAN de l'acheteur et montant rappelés sur l'écran de confirmation
| À faire
